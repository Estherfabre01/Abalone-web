📘 Fiche de Révision React — Version Détaillée

1. Composants

Un composant est une fonction qui retourne du JSX.

function Hello() {
  return <h2>Salut !</h2>;
}

function App() {
  return (
    <div>
      <Hello />
      <Hello />
    </div>
  );
}

2. Props

Les props permettent de passer des données d’un parent à un enfant.

function UserCard({ name, age }) {
  return <p>{name} a {age} ans.</p>;
}

<UserCard name="Esther" age={22} />

3. State (useState)

Le state permet de gérer des données internes au composant.

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Compteur : {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}

4. useEffect

Permet d’exécuter du code après le rendu (fetch, timers, écouteurs...).

useEffect(() => {
  console.log("Le composant est monté");

  return () => {
    console.log("Le composant est démonté");
  };
}, []);

Exemple avec API :

useEffect(() => {
  fetch("https://api.example.com/users")
    .then(res => res.json())
    .then(data => setUsers(data));
}, []);

5. useRef

Stocke une valeur persistante sans déclencher de re-render.

const inputRef = useRef();

function focusInput() {
  inputRef.current.focus();
}

<input ref={inputRef} />
<button onClick={focusInput}>Focus</button>

6. Rendu conditionnel

{isLogged ? <Dashboard /> : <Login />}

Version plus lisible :

if (!user) {
  return <Login />;
}
return <Dashboard user={user} />;

7. Listes et clés

const fruits = ["Pomme", "Banane", "Orange"];

<ul>
  {fruits.map((fruit, index) => (
    <li key={index}>{fruit}</li>
  ))}
</ul>

8. Formulaires (composants contrôlés)

const [email, setEmail] = useState("");

<input
  type="email"
  value={email}
  onChange={e => setEmail(e.target.value)}
/>

9. Context API

Permet de partager un état global sans passer les props partout.

const ThemeContext = createContext();

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Home />
    </ThemeContext.Provider>
  );
}

function Home() {
  const theme = useContext(ThemeContext);
  return <p>Thème actuel : {theme}</p>;
}

10. useReducer

Alternative à useState pour des états complexes.

function reducer(state, action) {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 };
    case "decrement":
      return { count: state.count - 1 };
    default:
      return state;
  }
}

const [state, dispatch] = useReducer(reducer, { count: 0 });

<button onClick={() => dispatch({ type: "increment" })}>+1</button>

11. useMemo & useCallback

useMemo : mémoriser une valeur

const expensiveValue = useMemo(() => compute(data), [data]);

useCallback : mémoriser une fonction

const handleClick = useCallback(() => {
  console.log("clic");
}, []);

12. Bonnes pratiques

  • Découper en composants simples et réutilisables.
  • Utiliser des clés stables (id plutôt qu’index).
  • Éviter les useEffect inutiles.
  • Préférer les composants fonctionnels.
  • Nommer clairement les états et handlers.
